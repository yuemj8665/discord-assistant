## 테스트 상세

### 음성 파이프라인 디버깅 과정

**문제 1 — VAD 프레임 크기 미스매치**
- Discord 표준 프레임(3840 bytes) → 다운샘플 후 640 bytes
- 코드가 30ms(960 bytes) 요구 → 640 < 960 → 모든 프레임 드롭
- 수정: VAD 프레임을 30ms → 20ms(640 bytes)로 변경

**문제 2 — asyncio 스레드 충돌**
- `write()` 메서드는 PacketRouter 스레드에서 실행
- `asyncio.create_task()` → `RuntimeError: no running event loop`
- 수정: `asyncio.run_coroutine_threadsafe()` 사용

**문제 3 — 침묵 감지 불가 (카운트 방식의 한계)**
- Discord는 사용자가 침묵하면 RTP 패킷 전송 중단
- `write()` 호출이 없으니 침묵 프레임 카운트 불가 → 발화 종료 미감지
- 수정: 100ms 주기 백그라운드 타이머로 `last_speech_time` 감시

**문제 4 — Whisper 내부 VAD 필터**
- 버퍼 내 zeros(OpusError 패치 데이터) 비중이 높아 Whisper가 무음으로 판단
- `VAD filter removed 00:00.900 of audio` → STT 결과 없음
- 수정: `vad_filter=False`, `no_speech_threshold=0.9`로 완화

**문제 5 — ffmpeg 미설치**
- `discord.FFmpegPCMAudio` 호출 시 `ClientException: ffmpeg was not found.`
- 수정: `brew install ffmpeg`

**문제 6 — TTS 파일 미삭제**
- ffmpeg 에러 발생 시 cleanup이 실행되지 않음
- 수정: try/finally로 cleanup 보장

**근본 문제 — Discord DAVE E2EE**
- Discord가 2026년 3월부터 모든 음성채널에 DAVE(종단간 암호화) 강제 적용
- 봇이 받는 오디오: DAVE 암호화됨 → OpusError 또는 노이즈 → STT 불가
- 봇이 보내는 오디오: 일반 PCM → 클라이언트가 DAVE 포맷 기대 → 무음
- DAVE는 서버 설정에서 끌 수 없음 (Discord 정책)
- 스테이지 채널은 DAVE 미적용이나 커뮤니티 서버 설정 필요 → 보류
